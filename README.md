# 若幸祭で使うレジシステムについて

## 構造について

<p>このレジシステムはWEBアプリケーションでの操作がメインとなります</p>
<p>また、データの管理としてgoogleが提供している「firebase realtimedatabase」を使用しています</p>
<p>商品を選択後、モーダルウィンドウが開くので個数を選択し、確定するとデータベースにタイムスタンプ付きでデータが格納されます</p>
<p>合計表示を押すとデータベースに格納されているデータを味ごとに合計しその販売数を出力します</p>
<p>在庫表示を押すとあらかじめ設定された在庫量と販売量の差分を出すことで、残り在庫数から販売可能予想数を出力します</p>
<p>レジが複数必要になるかわかりませんが、複数必要になるようだったら2相コミットを実装しなければいけないのでレジ1つがいいなぁと思っています（切実）</p>
